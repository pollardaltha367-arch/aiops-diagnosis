## 连接超时与网络链路

可能原因：目标服务未运行或端口未监听、DNS 解析错误、路由不可达或拥塞、防火墙或安全组阻断、连接池或超时设置不合理。

验证动作：

- 使用 `ping` 检查基础连通性，但不要把 ping 成功当作服务可用。
- 使用 `traceroute` 或 `tracert` 比较异常路径。
- 使用 `nc -vz host port`、`Test-NetConnection` 或等价工具验证端口。
- 检查服务状态、监听端口、防火墙和最近网络策略变更。

修复方向：恢复目标服务或端口监听；修复 DNS、路由或访问控制；调整连接池和超时配置。

## 网络丢包与端口异常

可能原因：链路拥塞、接口错包、网线或硬件故障、端口频繁 up/down、双工、速率、MTU 或路由配置不一致。

验证动作：

- 持续 ping 并记录丢包率和延迟分布。
- 查看端口 CRC、drop、error、up/down 计数。
- 对比带宽利用率、队列丢弃和设备日志。
- 更换链路或端口进行隔离测试。

修复方向：降低拥塞或重新分配流量；修复接口配置；切换端口、链路或故障硬件。

## CPU 与内存异常

可能原因：异常进程、死循环、流量突增、批处理、内存泄漏、缓存失控、并发过高、容量不足或版本回归。

验证动作：

- 使用 `top`、`htop`、`ps` 或任务管理工具定位高占用进程。
- 检查负载、CPU steal、上下文切换、内存趋势、swap 和 OOM。
- 对照请求量、发布记录和定时任务时间。
- 查看线程栈、GC 和应用错误日志。

修复方向：隔离异常进程并保存诊断信息；限流、扩容、回滚或优化代码；调整缓存、堆内存和资源限制。

## 数据库异常与慢查询

可能原因：数据库服务不可用、认证失败、连接池耗尽、网络异常、锁等待、慢 SQL、索引缺失或资源不足。

验证动作：

- 检查数据库服务、连接数、连接池和错误日志。
- 检查慢查询、执行计划、索引和锁等待。
- 对照数据库 CPU、IOPS、磁盘延迟和网络指标。
- 验证账号权限和连接字符串变更。

修复方向：恢复服务或修复认证、网络；调整连接池；优化 SQL、索引和事务；扩容或降低负载。

## 磁盘异常

可能原因：空间或 inode 耗尽、IO 饱和、坏盘、文件系统错误、日志无限增长、权限或挂载异常。

验证动作：

- 使用 `df -h`、`df -i` 检查容量与 inode。
- 使用 `iostat` 或平台指标检查延迟、队列和利用率。
- 检查内核、SMART、文件系统和挂载日志。
- 定位大文件、异常增长目录和权限变化。

修复方向：安全清理、归档或扩容；修复权限和挂载；更换故障磁盘并执行数据恢复流程。

## 服务不可用与执行失败

可能原因：进程退出、健康检查失败、端口冲突、配置错误、依赖异常、发布失败或权限限制。

验证动作：

- 检查进程、服务管理器、端口和健康检查。
- 查看崩溃日志、退出码和启动日志。
- 检查依赖服务、配置差异和最近发布。
- 验证账号、目录和文件权限。

修复方向：先保存现场，再恢复服务；修复配置、权限或依赖；回滚有证据指向的异常版本。

## 接口响应慢

可能原因：后端资源饱和、数据库慢查询、网络延迟、下游依赖响应慢或代码性能问题。

验证动作：

- 分解接口总耗时、应用耗时、数据库耗时和下游耗时。
- 查看吞吐量、错误率、P95/P99 延迟和线程池。
- 使用链路追踪定位最长 span。
- 对照流量、发布和依赖状态。

修复方向：优化慢路径、SQL 或依赖调用；增加缓存、限流、超时和熔断；扩容有明确瓶颈的服务。

## 配置、权限与依赖异常

可能原因：配置、环境变量、密钥或证书错误；账号权限变化；依赖服务、消息队列或外部 API 异常。

验证动作：

- 比较故障前后的配置差异和变更记录。
- 使用最小权限账号复现访问。
- 验证依赖健康、错误率、延迟和配额。
- 检查配置中心、密钥和证书有效期。

修复方向：回滚或修正已确认的配置变化；恢复正确权限与凭据；隔离、降级或切换异常依赖。

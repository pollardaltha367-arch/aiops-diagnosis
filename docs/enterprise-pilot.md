# 企业日志初筛试点指南

## 适用任务

本项目适合承担以下低风险任务：

- 在本地或隔离环境读取导出的日志文件。
- 将重复异常日志聚合为模板簇。
- 将已知故障与未知严重事件分开。
- 生成带评分依据的高、中、低优先级人工复核队列。
- 为值班人员提供证据、缺口和下一步只读检查建议。

它不负责自动确认根因、连接生产系统或执行修复。

## 最小试点范围

建议选择一个非核心服务最近7天的脱敏日志，每次按15分钟或1小时窗口导出，单文件不超过50MB。试点前由企业运维人员提供：

- 日志字段说明与时间范围。
- 已知正常模板或常见噪声。
- 5至20个已复盘故障及真实结论。
- 不允许离开企业环境的敏感字段清单。

## 执行

```powershell
python3 scripts/triage_file.py incident.csv --output triage-output
```

输出：

- `report.md`：诊断与安全边界。
- `review_queue.csv`：人工复核队列，可导入Excel或工单系统。
- `result.json`：结构化结果，便于后续系统集成。

## 人工复核流程

1. 先处理高优先级模板，再处理中优先级。
2. 检查评分理由、出现次数、涉及对象和样例证据。
3. 结合指标、变更记录、拓扑和上下文确认是否为真实异常。
4. 在复核表中记录“真实异常/正常噪声/证据不足”。
5. 只有企业运维人员确认后，才能进入现有处置流程。

## 建议验收指标

- 告警召回率和误报率。
- 高、中优先级队列覆盖的真实事故比例。
- 原始日志行数到复核模板数的压缩率。
- 每个复核项的平均处理时间。
- 未知严重事件中最终确认故障的比例。
- 敏感字段抽查漏脱敏数量。

不要提前设定固定准确率。第一轮先建立企业自己的基线，再根据误报明细调整正常模板和阈值。

## 安全要求

- 默认只在企业本地或隔离环境运行。
- 原始日志不上传第三方服务。
- 不将基础脱敏描述为合规认证。
- 不自动执行命令、重启、配置修改或数据删除。
- 输出文件按企业现有日志权限管理。
